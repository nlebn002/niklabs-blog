# Conventions

- Prefer existing naming, file placement, and dependency patterns.
- Keep comments sparse and useful; explain non-obvious decisions, not syntax.
- Do not add mocks for behavior that should be covered through real module boundaries or integration tests.
- Avoid unrelated refactors while implementing a feature.
- Keep secrets out of committed files.
- Treat `.ai/context/` as the canonical project context for Claude and Codex.
