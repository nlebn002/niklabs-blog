# Frontend

Follow the existing frontend layout under `frontend/`.

Implementation defaults:
- Keep feature code in the closest existing feature-oriented location.
- Use TanStack Query for server state where the app already does.
- Use the generated Orval client instead of hand-written API calls.
- Preserve the custom fetch behavior, including CSRF and cookie handling.
- Keep UI components consistent with the existing styling and interaction patterns.

Frontend reference: `frontend/README.md`
